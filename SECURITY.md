# Security Policy

| Version | Supported          |
| ------- | ------------------ |
| 5.1.x   | :white_check_mark: |
| 5.0.x   | :x:                |
| 4.0.x   | :white_check_mark: |
| < 4.0   | :x:                |

---

## Reporting a Vulnerability

If you discover a security vulnerability within this project, please report it responsibly and privately.

### How to Report

Please report vulnerabilities through one of the following methods:

- GitHub Security Advisories
- Private maintainer email
- Official support contact channel

> Please do not create public GitHub issues for security-related reports.

---

### What to Include

When submitting a vulnerability report, please include:

- A clear description of the vulnerability
- Steps to reproduce the issue
- Affected versions or environments
- Potential impact of the vulnerability
- Screenshots, logs, or proof-of-concept code (if applicable)

Providing detailed information helps us investigate and resolve issues faster.

---

### Response Timeline

We aim to respond to security reports as quickly as possible.

| Stage | Expected Response Time |
| ------ | ---------------------- |
| Initial Response | Within 48 hours |
| Status Update | Within 3-5 business days |
| Resolution Time | Depends on severity and complexity |

You will receive updates throughout the investigation process.

---

### Responsible Disclosure

We kindly ask that you:

- Avoid public disclosure before a fix is released
- Do not access or modify other users’ data
- Avoid actions that could disrupt services or compromise user privacy
- Act in good faith to help improve project security

---

### Security Updates

If the vulnerability is confirmed:

- A security patch or fix will be developed
- Supported versions will receive updates
- A public security advisory may be published after resolution
- Credit may be given to the reporter (if requested)

If the report is determined to be invalid or out of scope, we will communicate the reason clearly.

---

Thank you for helping improve the security and reliability of this project.
